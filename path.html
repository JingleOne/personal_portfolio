The essence of this project

How A* search algorithm works

The algorithm uses heuristic function combine with cost to determine the next node to expand. A heuristic is extra information the algorithm uses along side with what it is already provided. For example, when this program is finding the shorting path to the goal, it will uses the straight line distance from the start to the goal as heuristic function to determine the general "direction" to explore instead of blindly expanding all the neighboring nodes to brute force the solution.

In the top diagram, A is the start/B is the goal and the neighboring grids of A are expanded. The number on the top left of all the green squares represent the step cost. It cost 10 to go one step horizontally and vertically, 14 to go diagonally. The top right number represent the heuristic function, which is the straight line distance to the goal. Add these two numbers up, we will get the total cost. Since the grid with 42 is the lowest cost, we will explore it first, therefore marking it red.
